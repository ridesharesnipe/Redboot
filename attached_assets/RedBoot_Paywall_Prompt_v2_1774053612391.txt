PAYWALL IMPLEMENTATION — RED BOOT'S SPELLING ADVENTURE
=======================================================

IMPORTANT: Do not change any existing UI, styling, components, or functionality.
Only ADD the paywall system described below. Do not refactor or "improve" anything else.
Do not touch: Onboarding, ChildSetup, Landing, Dashboard, PhotoCapture, character components, or any game UI.

This is a Stripe-powered paywall for the web app. Not Apple IAP.


== THE FLOW ==

Current: onboarding → child setup → dashboard → unlimited practice for free.

New: onboarding → child setup → character select → dashboard → upload words → ONE free practice session → PAYWALL → everything locked until they subscribe.

No paywall appears BEFORE the first practice session. The user experiences the app first, the kid plays the pirate game, the parent sees it work, THEN we ask for money. This is intentional — the paywall hits at the moment of peak excitement, not before they've tried it.


== PRICING (read carefully — these are intentional non-round numbers) ==

Do NOT "fix" these to .99 numbers. The non-round pricing is a deliberate conversion tactic —
$3.33/month feels like a real calculated price, not a marketing trick. Users have built
defensibility against .99 pricing. Non-round numbers feel more authentic and trustworthy.

Standard plans (with 7-day free trial):
  - Annual:  $39.96/year  (displayed as "$3.33/month" on the pricing card)
  - Monthly: $6.87/month

No-trial plans (10% discount for skipping trial):
  - Annual:  $35.88/year  (displayed as "$2.99/month" on the pricing card)
  - Monthly: $6.18/month

Transaction abandonment offer (no trial, hard discount):
  - Annual:  $23.88/year  (displayed as "$1.99/month", shown as "60% off")

The yearly strikethrough price: $82.44 (that's $6.87 x 12 — what monthly would cost for a year)


== SUBSCRIPTION STATE ==

Add subscription state to localStorage:

Key: "redboot-subscription"
Value: JSON object:
{
  "status": "free" | "trial" | "active" | "cancelled",
  "freeSessionUsed": false | true,
  "stripeCustomerId": null | "cus_xxx",
  "subscriptionId": null | "sub_xxx",
  "currentPeriodEnd": null | "2026-04-20T00:00:00Z",
  "plan": null | "monthly" | "annual"
}

On first install, status is "free" and freeSessionUsed is false.
After the first practice session completes (when SimplePractice shows results), set freeSessionUsed to true.

== STRIPE INTEGRATION ==

Use Stripe Checkout (hosted page) for payment. Keeps PCI compliance simple.

Stripe prices to create (create these in Stripe dashboard, store IDs as env vars):
  STRIPE_PRICE_MONTHLY            — $6.87/month, 7-day free trial
  STRIPE_PRICE_ANNUAL             — $39.96/year, 7-day free trial
  STRIPE_PRICE_MONTHLY_NO_TRIAL   — $6.18/month, no trial
  STRIPE_PRICE_ANNUAL_NO_TRIAL    — $35.88/year, no trial
  STRIPE_PRICE_ANNUAL_ABANDONMENT — $23.88/year, no trial

Server endpoints:

POST /api/create-checkout-session
  Body: { plan: "monthly" | "annual", trial: boolean, offer: "standard" | "abandonment", email: string, deviceId: string }
  - Look up or create Stripe customer by email
  - Create Checkout Session with appropriate price ID
  - If trial is true, set subscription_data.trial_period_days: 7
  - Set success_url: /dashboard?subscription=success
  - Set cancel_url: /dashboard?checkout=cancelled
  Returns: { url } — redirect user to this Stripe Checkout URL

POST /api/webhook
  Stripe webhook endpoint. Verify signature with STRIPE_WEBHOOK_SECRET.
  Handle events:
    checkout.session.completed:
      - Find device by metadata.deviceId
      - Store stripeCustomerId, subscriptionId in database
      - Set status to "trial" (if trial_period) or "active" (if no trial)
      - Set currentPeriodEnd from subscription data
      - Set plan to "monthly" or "annual"
    customer.subscription.updated:
      - Update status and currentPeriodEnd
    customer.subscription.deleted:
      - Set status to "cancelled"
    invoice.payment_failed:
      - Set status to "past_due"

GET /api/subscription-status?deviceId=xxx
  Returns subscription state for the given device
  Since there's no auth, tie subscription to a device UUID:
    - Generate UUID on first visit, store as "redboot-device-id" in localStorage
    - Send deviceId with all API calls
    - Server stores deviceId → subscription mapping in database

On app load, call GET /api/subscription-status and sync result to localStorage "redboot-subscription".
This way the app works offline (reads localStorage) but stays in sync when online.

Success flow:
  - User completes Stripe Checkout → redirected to /dashboard?subscription=success
  - App detects ?subscription=success query param
  - Shows a brief celebration: "Welcome aboard, Captain! Full access unlocked."
  - Updates localStorage subscription status
  - Removes all paywall gates

Cancel/abandonment flow:
  - User closes Stripe Checkout → redirected to /dashboard?checkout=cancelled
  - App detects ?checkout=cancelled query param
  - Sets "redboot-checkout-abandoned": true in localStorage
  - On next user action, check this flag and show Transaction Abandonment paywall

== DESIGN RULES ==

- All paywall screens: clean, minimal, parent-focused design
- NO pirate theme on paywalls — the paywall talks to the adult, not the kid
  (Exception: transaction abandonment can use "Wait, Captain!" heading)
- Font: system font stack or whatever the app uses for non-game UI
- Colors:
    Blue (#378ADD) — info, highlights, selected plan border
    Green (#1D9E75) — success, CTA on abandonment offer, trial toggle ON
    Purple (#534AB7) — primary CTA buttons, section headers
    Amber (#FAEEDA) — warning backgrounds, timeline step 2
- The paywall should feel like a premium product page, not a popup ad
- Mobile-first — must look good on phone screens (this is primarily a mobile web app)
- Smooth transitions: paywall slides up from bottom or fades in, not a hard cut
- "Maybe later" / dismiss option must ALWAYS be visible — never hide the exit
- COPPA: No data collection from the child. Email is collected from the parent only at checkout time.
